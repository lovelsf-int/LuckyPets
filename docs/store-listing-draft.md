# LuckyPets Store Listing Draft / 商店资料草稿

## App Identity / 应用身份

- App name / 应用名：LuckyPets
- Subtitle / 副标题：宠物交友与安全匹配
- Category / 分类：Lifestyle / 生活
- Age rating posture / 年龄分级口径：owner-facing pet social app; no dating between people; moderation and reporting are required.

- 年龄分级口径：面向宠物家长的宠物社交应用，不做人与人的约会定位；必须提供审核、举报和安全处理能力。

## Short Description / 简短描述

Find safer social, playdate, and responsible breeding conversations for pets, with owner-first boundaries and review gates.

为宠物寻找更安全的交友、玩伴和负责任繁育沟通，强调家长边界和审核门槛。

## Long Description / 完整描述

LuckyPets helps pet owners create pet profiles, browse compatible pets, match by intent, and start structured conversations. Social and playdate matches focus on owner consent, pet temperament, health context, and safe meetup preparation.

LuckyPets 帮助宠物家长创建宠物资料、浏览合适的新朋友、按目的匹配，并开启结构化沟通。交友和玩伴匹配强调家长同意、宠物性格、健康背景和安全见面准备。

Breeding conversations are treated as a restricted flow. Before breeding can be treated as approved, owners should provide required evidence such as identity verification, vaccination records, age and body condition checks, and relevant genetic screening. Reports, blocking, unmatching, account deletion, and moderation review are core safety features.

繁育沟通被视为受限制流程。繁育被视为可推进前，家长应提供身份核验、疫苗记录、年龄与体况证明，以及相关基因筛查等材料。举报、拉黑、解除匹配、账号删除和审核队列是核心安全能力。

## Review Notes / 审核备注

- LuckyPets is not a human dating app.
- Breeding intent is gated by eligibility review and safety copy.
- Users can report conversations, block owners, unmatch, and request account deletion.
- Medical records are private by default and should not be publicly exposed.
- Location should remain coarse by default; precise home addresses are not part of public profiles.

- LuckyPets 不是真人约会应用。
- 繁育意图受资格审核和安全文案约束。
- 用户可以举报会话、拉黑家长、解除匹配并申请删除账号。
- 医疗记录默认私密，不应公开展示。
- 位置默认使用粗粒度区域，公开资料不包含精确家庭地址。

## Privacy And Data Safety / 隐私与数据安全

Draft disclosure areas:

- Account data: email, display name, active pet profile.
- User content: pet profile text, photos, health record metadata, messages, reports.
- Approximate location: city or coarse area only.
- Diagnostics: crash logs and basic analytics when enabled.
- Deletion: account deletion request must be available in-app.

草稿披露范围：

- 账号数据：邮箱、展示名、当前宠物资料。
- 用户内容：宠物资料文本、照片、健康记录元数据、消息和举报。
- 大致位置：仅城市或粗粒度区域。
- 诊断数据：启用后可收集崩溃日志和基础分析。
- 删除能力：App 内必须提供账号删除请求。
